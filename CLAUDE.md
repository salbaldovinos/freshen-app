## What this app does

Mobile app (iOS + Android) for tracking livestock breeding cycles. Users record pairings, track gestation countdowns, confirm pregnancies, and log births. Replaces generic countdown-timer apps used by small-scale goat and livestock breeders.

## App identity

- **App name:** Freshen
- **Bundle ID:** com.freshenapp.freshen
- **App Store subtitle:** Livestock Breeding & Due Date Tracker
- **Tagline:** Every new life starts here.
- **Source of truth for all branding strings:** `constants/app.ts` — NEVER hardcode the app name anywhere else.
- **Dark mode:** Not supported in v1.0. Light mode only.

## Name-change policy

The app name, tagline, bundle ID, and Store metadata are isolated in `constants/app.ts`. If the name changes, only that file and `app.json` need updating. To enforce this:
- Import APP_NAME from `constants/app.ts` wherever the name appears in UI
- Never write the string "Freshen" directly in component code
- App Store copy (screenshots, descriptions) is the only place the name lives outside code

- expo-font (Cormorant + DM Sans custom fonts)

## Code conventions

- Use early returns for error conditions (never deeply nested ifs)
- All user-facing strings go in `constants/strings.ts` — never hardcode UI copy
- **The app name must always be imported from `constants/app.ts` as `APP_NAME` — never write the string "Freshen" directly in component or screen code**
- **ALL date math lives exclusively in `lib/gestation.ts`** — use date-fns functions (differenceInCalendarDays, addDays, parseISO). Never use raw `new Date()` arithmetic anywhere.
- `dueDate` is always derived, never stored. Compute via `calculateDueDate(pairingDate, gestationDays)`. Sort by due date in-memory after computing.
- **Generate IDs with `Crypto.randomUUID()` from `expo-crypto`** — never the global `crypto`, which does not exist in Hermes release builds (dev builds mask this; it shipped as every-insert-fails until E2E caught it)
- **jest is pinned to `~29.7.0`** — jest-expo 55 targets jest 29 internals; jest 30 breaks component-test suite loading. Don't bump without bumping Expo.
- Zod schemas defined in `lib/schemas.ts` and imported where needed
- All tier-gating logic lives in `lib/tierChecks.ts` — no tier checks in UI components
- Follow patterns in existing query files when adding new DB queries
- Every async operation must handle loading, success, and error states (4 states: empty, loading, populated, error)
- All PostHog event tracking calls go through `lib/analytics.ts` — never call PostHog directly in components
- Color tags use design system values: `'gray' | 'ember' | 'orange' | 'yellow' | 'green' | 'teal' | 'blue' | 'purple'`
- Brand voice: no emoji in UI or notifications. Sentence case everywhere. Short confirmations. See design system Section 5.

## Business rules

- **Animal counting:** "Animals" = distinct `animalName` values (case-insensitive) across non-archived breeding records. No separate animals table.
- Free tier: max 10 animals. Paid tier: unlimited.
- Free tier: 1 active push notification. Paid tier: unlimited.
- Free tier: local SQLite only. Paid tier: PowerSync cloud sync.
- Free tier: goats only. Paid tier: all 7 species.
- **Free for all tiers:** birth logging (full create), color tag customization, pregnancy confirmation, gestation tracking.
- Goat gestation default = 150 days. All species configurable via SPECIES_CONFIG.
- "Days bred" = calendar days from pairing date to today.
- Due date = pairing date + species gestation days (calculated, not stored).
- Confirmed pregnancy changes status from "Bred" → "Pregnant" (yellow → green).
- "Overdue" = today > due date AND no birth logged.
- One breeding record can have multiple birth log entries.
- Birth notes: max 300 characters. Breeding notes: max 500 characters.
- 7-day free trial only shown when user hits a paywall, not on first install.

## E2E testing notes (learned 2026-07-17, keep flows consistent with these)

- Maestro needs a Java runtime: `brew install openjdk`, then export `JAVA_HOME`/`PATH` from `/usr/local/opt/openjdk`.
- **Run flows against a Release build** (`npx expo run:ios --configuration Release`). Dev-client builds break `launchApp: clearState` (the cleared app loses its Metro connection) and hide release-only bugs.
- Every flow starts with `launchApp: clearState: true` — SQLite persists otherwise and flows contaminate each other.
- `BreedingCard` is one flattened iOS accessibility element (its Pressable concatenates all card text). Match card content with substring regexes (`.*Daisy.*`), never exact text. Standalone texts (headers, sheet items, form labels) match exactly.
- After a save, wait for the home header count (`extendedWaitUntil: visible: "N active breedings."`) before asserting — the typed name also matches the form input, and plain asserts time out during the save→navigate→refetch transition.
- Dismiss the keyboard with an explicit `swipe` from `50%, 40%` to `50%, 15%` (form ScrollViews have `keyboardDismissMode="on-drag"`). Maestro's `hideKeyboard` is flaky on iOS; a centered swipe lands on the keyboard itself.

## Do NOT modify

- `components/ui/` — react-native-reusables components; re-copy from source if updates needed. Exception: `sheet.tsx` is custom-written (no upstream source); its two wrapper Pressables must keep `accessible={false}` or iOS flattens the sheet and VoiceOver/E2E cannot reach the items
- `db/migrations/` — never edit manually; always use `drizzle-kit generate`
- `_discovery/` — reference documents, not code
